\# Exercice 2 – Analyse DevOps d'une organisation



\## Organisation

\*\*Netflix\*\* est une entreprise américaine de streaming vidéo et de production de contenu. Elle est reconnue pour son adoption avancée des pratiques DevOps et du cloud computing.



\## Pratiques DevOps identifiées

\- Déploiement continu (Continuous Delivery)

\- Infrastructure as Code (IaC)

\- Observabilité et monitoring avancé

\- Culture du “Freedom \& Responsibility” pour l’automatisation des processus

\- Boucles de feedback rapides via tests automatisés et chaos engineering



\## Outils utilisés

\- Spinnaker pour le Continuous Delivery

\- Jenkins pour CI/CD

\- GitHub pour le versionning

\- AWS pour l’infrastructure cloud

\- Chaos Monkey pour le test de résilience

\- Prometheus et Grafana pour le monitoring



\## Points forts et leçons apprises

\- L’automatisation réduit les erreurs humaines et accélère le déploiement

\- L’infrastructure as code rend la gestion scalable et reproductible

\- Le monitoring et les boucles de feedback permettent de détecter rapidement les incidents

\- Encourager l’autonomie et la responsabilité des équipes améliore l’innovation
